Problem

Enterprise presales teams operate over heterogeneous document corpora — PDFs, PPTs, DOCX, Excel — where queries span multiple document types and intents. Standard RAG retrieves indiscriminately: a pricing question gets the same retrieval path as a technical spec question, degrading precision. At 8K token context limits, irrelevant retrievals crowd out relevant ones. The problem: build a RAG system that routes by intent, expands queries systematically, and stays within token budgets without sacrificing coverage.

Architecture

Four-layer system. The ingestion layer handles PDF/PPT/DOCX/Excel parsing with format-aware chunking. The query layer applies four-strategy expansion (synonym, decomposition, hypothetical document, step-back) before retrieval. The routing layer uses a seven-category intent classifier to select document-type preference weights — a pricing query preferentially retrieves from Excel and structured docs; a technical query from PDFs and DOCX. The context layer enforces token-aware windowing at the 8K limit, ranking and truncating chunks by relevance score before assembly.

Implementation — Key Components

Seven-Category Intent Classifier

Fine-tuned classifier mapping query intent to document-type preference weights. Categories cover: pricing, technical specification, compliance, competitive, timeline, personnel, general. Preference weights shift retrieval scoring rather than hard-filtering — all document types remain accessible.

Four-Strategy Query Expansion

Parallel expansion generating synonym variants, decomposed sub-questions, hypothetical answer documents (HyDE), and step-back abstractions. All four variants are retrieved against; results are merged and deduplicated by embedding similarity before ranking.

Token-Aware Context Windowing

Chunk ranker that scores retrieved segments by relevance, then greedily fills the 8K context window. Stops before overflow rather than truncating mid-chunk, preserving coherence.

Security-First Test Suite

150+ test cases across 13 files covering adversarial queries, injection attempts, and privilege-escalation patterns. Surfaced 3 critical CVEs pre-deployment with zero breaking changes to the live system.
